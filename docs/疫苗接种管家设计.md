# 疫苗接种管家功能扩展设计方案

## 一、数据结构

### 现有模型
- `VaccinePlanTemplate` - 疫苗模板（2021国家免疫规划标准）
- `BabyVaccineSchedule` - 宝宝的疫苗接种记录

### 核心字段
```
BabyVaccineSchedule:
- id, babyId, templateId
- vaccineType (一类/二类)
- vaccineName (疫苗名称)
- ageInMonths (接种月龄)
- doseNumber (剂次)
- vaccinationStatus (pending/completed/missed/skipped)
- vaccineDate (实际接种日期)
- hospital, batchNumber, doctor, reaction, note
```

## 二、功能模块

### 1. 即将接种（计算逻辑）
**根据宝宝出生日期 + 2021国家免疫规划自动计算**

| 疫苗 | 剂次 | 接种月龄 |
|------|------|----------|
| 乙肝 | 3针 | 0、1、6月龄 |
| 卡介苗 | 1针 | 0月龄(出生) |
| 脊灰(口服) | 4剂 | 2、3、4月龄，4岁 |
| 百白破 | 4针 | 3、4、5月龄，18月龄 |
| 麻腮风 | 2针 | 8、18月龄 |
| 甲肝 | 1针 | 18月龄 |
| 乙脑(减毒) | 2剂 | 8月龄，2岁 |
| A群流脑 | 2针 | 6、9月龄 |
| A+C群流脑 | 2针 | 3岁，6岁 |
| 白破 | 1针 | 6岁 |

**计算规则**：
- 根据 `birthDate` 计算当前月龄 = floor((today - birthDate) / 30)
- 找出所有 `ageInMonths > currentMonth` 的待接种疫苗
- 按月龄排序，显示距离最近的5-8个

### 2. 已接种记录（CRUD）
**支持增删改查**

- **查看**：列表展示，支持分页
- **新增**：手动添加接种记录（医院、批号、医生、反应等）
- **修改**：编辑已接种记录信息
- **删除**：删除接种记录（确认提示）

**表格字段**：
- 疫苗名称
- 接种日期
- 剂次
- 医院
- 批号
- 医生
- 反应
- 操作（编辑/删除）

### 3. 联动同步
**接种登记后自动更新**

- 已接种 → 从"待接种"列表移除，加入"已接种记录"
- 下次预计 → 自动更新为下一个待接种疫苗
- 数据结构 → `vaccinationStatus: 'completed'`

### 4. 下次预计显示
**显示格式**：

```
下次预计：乙肝疫苗第2针 (6月15日)
或
下次预计：暂无计划
```

## 四、API 设计

### 获取疫苗列表（含计算）
```
GET /api/baby/:babyId/vaccines
Response: {
  upcoming: [...],  // 自动计算的待接种
  completed: [...],  // 已接种记录
  nextVaccine: {...} // 下次预计
}
```

### 标记接种
```
POST /api/baby/:babyId/vaccines
{ action: 'complete', templateId, vaccineDate, hospital, ... }
```

### CRUD 操作
```
POST /api/baby/:babyId/vaccines   // 新增
PUT /api/baby/:babyId/vaccines/:id  // 修改
DELETE /api/baby/:babyId/vaccines/:id // 删除
```
