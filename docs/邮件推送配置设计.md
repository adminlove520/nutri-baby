# 邮件推送配置设计方案

## 一、现状问题

1. **定时任务区域有重复的开关** — 消息通知区和定时任务区的开关重复
2. **没有独立的邮件配置** — 只有开关，没有配置和测试
3. **Cron API 有鉴权问题** — 注释掉了鉴权

---

## 二、设计方案

### 2.1 消息通知设置（保留现有，简化）

| 开关 | 功能 |
|------|------|
| 站内消息 | 应用内通知提醒 |
| 邮件通知 | 是否接收邮件（总开关）|

### 2.2 定时任务设置（新增独立分组）

| 开关 | 功能 | 触发方式 |
|------|------|----------|
| 疫苗接种提醒 | 接种前1天站内+邮件提醒 | Vercel Cron 每日触发 |
| 每日AI育儿锦囊 | 每天一则科学育儿建议 | Vercel Cron 每日触发 |
| 自动执行定时任务 | 总开关，控制是否执行 | Vercel Cron |

### 2.3 邮件推送配置（新增独立分组）

| 功能 | 说明 |
|------|------|
| 当前邮箱 | 显示已配置的接收邮箱 |
| 测试邮件通道 | 点击发送测试邮件 |
| 发送状态 | 显示上次发送时间/状态 |

---

## 三、API 设计

### 3.1 新增邮件测试 API（需要登录）

```
POST /api/user/test-email
Response: { success: true, message: '测试邮件已发送' }
```

### 3.2 新增定时任务手动触发 API（需要登录）

```
POST /api/user/trigger-notify
Body: { type: 'vaccine' | 'aiTip' | 'all' }
Response: { success: true, count: 5 }
```
